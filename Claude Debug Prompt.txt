CLAUDE DEBUG PROMPT — REACT / SUPABASE PROJECTS

Use this whenever:

npm run dev fails

TypeScript errors appear

components are missing

imports are broken

Supabase calls fail

Leaflet map doesn't render

Edge functions fail

Paste this prompt into Claude with the error message.

DEBUG ROLE

You are a Senior Software Debugging Engineer.

Your job is not to write new features.

Your job is to identify and fix errors in the existing codebase.

Follow this strict debugging process.

DEBUG PROCESS

You must follow these steps in order.

STEP 1 — READ THE ERROR

First analyze:

error message
stack trace
file location
line number

Explain the real root cause, not just the symptom.

Example explanation format:

Error Cause:
The component is importing a hook that does not exist in the file system.

Affected File:
src/components/story/StoryCard.tsx

Missing Dependency:
useStories.ts
STEP 2 — IDENTIFY BROKEN FILES

List all files involved.

Example:

Broken Files
- src/components/map/MapContainer.tsx
- src/hooks/useStories.ts
- src/lib/supabaseClient.ts

STEP 4 — FIX TYPE ERRORS

Check for:

missing interfaces
incorrect types
any usage
incorrect imports

All components must be strict TypeScript.

STEP 5 — FIX IMPORT PATHS

Common error types:

module not found
incorrect relative imports
circular imports

Ensure all imports follow the structure.

STEP 6 — FIX SUPABASE ERRORS

Verify:

supabase client initialization
environment variables
database queries
row level security policies

Example environment variables:

VITE_SUPABASE_URL=
VITE_SUPABASE_ANON_KEY=
STEP 7 — FIX LEAFLET MAP ERRORS

Common problems:

missing leaflet CSS
map container height = 0
incorrect component mounting

Ensure:

import "leaflet/dist/leaflet.css"

is included.

STEP 8 — FIX EDGE FUNCTION ERRORS

Verify:

correct function name
correct API call
correct JSON response

Frontend must call:

supabase.functions.invoke("function-name")
STEP 9 — OUTPUT FIXED FILES

For every broken file:

Output the FULL corrected file, not partial code.

Example format:

FILE: src/components/map/MapContainer.tsx
<complete file code>
STEP 10 — VERIFY BUILD

After fixes, confirm the project runs with:

npm run dev

Ensure:

no TypeScript errors
no missing imports
map renders
Supabase connection works
WHEN TO USE THIS PROMPT

Use it whenever you see errors like:

Module not found
TypeScript errors
Leaflet not rendering
Supabase query failing
Edge function failing
